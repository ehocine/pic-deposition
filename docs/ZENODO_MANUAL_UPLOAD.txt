Zenodo 504 workaround — manual upload (no GitHub sync)
======================================================

If https://zenodo.org/account/settings/github/ shows "504 Gateway Timeout"
when you click Sync, use a MANUAL deposition instead. This is official and
equally valid for CPC submission.

Option A — Retry GitHub sync (quick tries)
------------------------------------------
1. Wait 1–2 hours (Zenodo outages are often transient).
2. Zenodo → Profile → Linked accounts → Disconnect GitHub → Connect again.
3. GitHub settings → enable ONLY pic-deposition (not bulk sync of all repos).
4. Click Sync once, wait 60s, refresh the page (sync may finish in background).

Option B — Manual upload (recommended if 504 persists)
----------------------------------------------------
1. Log in at https://zenodo.org/
2. Click "+ New upload" (top right).
3. Upload type: Software
4. Upload file: pic-deposition-v1.0.0-source.zip (create below)
5. Fill metadata (copy from CITATION.cff / .zenodo.json):
   - Title: PIC Deposition Benchmark Suite: A Reproducible Reference for 2D
     Electrostatic PIC Charge Deposition
   - Authors: Hocine, Elhadj — Independent Researcher
   - Description: (from .zenodo.json "description" field)
   - License: MIT
   - Version: 1.0.0
   - Related identifier: https://github.com/ehocine/pic-deposition (Relation: is supplement to / is identical to)
6. Click Publish → copy DOI (10.5281/zenodo.XXXXXXX)

Create the zip locally:
  cd /Users/elhadj/Desktop/Paper
  bash scripts/make_zenodo_archive.sh
  → produces dist/pic-deposition-v1.0.0-source.zip

Option C — Submit CPC without DOI first
---------------------------------------
Many CPC submissions accept:
  "Software available at https://github.com/ehocine/pic-deposition (Zenodo DOI pending)."
Add the DOI in proof stage when Zenodo works.
