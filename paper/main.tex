\title{Cache-Aware and GPU-Accelerated Charge Deposition for Electrostatic Particle-In-Cell Simulations}

\author{\IEEEauthorblockN{Elhadj Hocine, PhD}
\IEEEauthorblockA{\textit{Independent Researcher}\\
Email: hocineelhadj@gmail.com}}

\maketitle

\begin{abstract}
Charge deposition is a dominant cost in explicit Particle-In-Cell (PIC) plasma simulations because it performs irregular, scatter-oriented updates to a structured grid.
This work presents a reproducible two-dimensional electrostatic PIC framework and evaluates Nearest Grid Point (NGP), Cloud-In-Cell (CIC), Triangular-Shaped Cloud (TSC), and Esirkepov charge-conserving deposition under cache-aware CPU optimizations.
We compare Array-of-Structures (AoS) and Structure-of-Arrays (SoA) layouts, optional cell sorting, and OpenMP parallelization with thread-local grid privatization.
Benchmarks measure sort and deposit time separately, effective memory bandwidth, roofline arithmetic intensity, charge conservation error, energy drift, a 2000-step conservation study, spectral grid noise, and CPU strong scaling on particle counts from $10^4$ to $10^6$.
Separate CPU results on Apple M1 Pro and GPU results on Tesla T4 (same-host CPU baseline) are reported to avoid misleading cross-platform comparisons.
A Langmuir-wave validation confirms scheme-independent oscillation frequencies within 5\% across all four deposition methods.
On Tesla T4, atomic CUDA deposition achieves $5$--$8\times$ speedup over single-threaded host CPU at $N_p=10^5$--$10^6$; multi-block privatized tiles preserve charge conservation but remain slower than atomics due to per-tile full-particle scans.
Roofline analysis shows deposition is memory-bandwidth limited; CIC offers the best accuracy--performance trade-off on CPU, while GPU backends favor large $N_p$.
Source code and archived CSVs are available at \url{https://github.com/ehocine/pic-deposition} (Zenodo DOI: \texttt{TODO-DOI}).
\end{abstract}
